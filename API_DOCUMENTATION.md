# 📚 Kiddbill Suite - Comprehensive API Documentation

เอกสารนี้อธิบายการทำงานของ API แต่ละระบบ (Domain) ว่ามีหน้าที่อะไร และถูกนำไปใช้งานในขั้นตอนไหนของแอปพลิเคชัน (Best Practice สำหรับการส่งมอบให้ Frontend/Mobile Developer)

> [!TIP]
> **Base URL:** `http://localhost:3002/v1`
> **Authorization:** ขาที่ต้องใช้สิทธิ์ ให้แนบ Header `Authorization: Bearer <Clerk_Token>` เสมอ

---

## 1. 🧾 ระบบจัดการบิล (Bills)
**หน้าที่:** เป็นแกนหลักของระบบ ใช้สำหรับสร้างบิลตั้งต้น และจัดการสถานะของบิล (เปิด/ปิด)

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/bills` | **สร้างบิลใหม่** รับข้อมูลเบื้องต้นเช่น ชื่อบิล (Title), ค่า Service Charge, หรือตั้งค่า VAT เบื้องต้น |
| `GET` | `/bills` | **ดึงรายการบิลทั้งหมด** ของ User ที่กำลัง Login อยู่ (เอาไปทำหน้ารวมบิล History) |
| `GET` | `/bills/:id` | **ดึงข้อมูลบิลแบบเจาะจง** (ดึงบิลเดียว) เพื่อเอาไปแสดงในหน้ารายละเอียดบิล |
| `PATCH` | `/bills/:id` | **แก้ไขข้อมูลบิล** เช่น เปลี่ยนชื่อบิล หรือแก้ไขอัตรา VAT / Service charge |
| `PATCH` | `/bills/:id/close` | **ปิดบิล (Finalize)** เมื่อกดปุ่มนี้ ระบบจะล็อกไม่ให้เพิ่มรายการอาหารอีก และพร้อมเข้าสู่โหมดเรียกเก็บเงิน |
| `GET` | `/bills/:id/summary` | **สรุปยอดเงิน (Summary)** เรียกดูยอดรวม (Total), ยอดของแต่ละคน (Subtotals) ที่ระบบคำนวณให้แบบสำเร็จรูป |
| `DELETE`| `/bills/:id` | **ลบบิล** ลบบิลทิ้ง (พร้อมกับ Items และ Members ที่เกี่ยวข้องทั้งหมด Cascade Delete) |

## 2. 🍔 ระบบรายการอาหาร (Bill Items)
**หน้าที่:** จัดการรายการของที่สั่ง (Items) ในบิลแต่ละบิล

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/bill-items` | **เพิ่มรายการอาหาร** เช่น "ส้มตำ 50 บาท" โดยต้องแนบ `billId` ไปด้วยว่าเพิ่มเข้าบิลไหน |
| `PATCH` | `/bill-items/:id` | **แก้ไขรายการอาหาร** เช่น แก้ไขราคา หรือเปลี่ยนชื่อรายการ |
| `DELETE`| `/bill-items/:id` | **ลบรายการอาหาร** |

## 3. 👥 ระบบสมาชิกในบิล (Bill Members)
**หน้าที่:** จัดการคนที่จะเข้ามาหารค่าใช้จ่ายในบิลนั้นๆ ทั้งคนที่มีบัญชี (User) และคนที่ไม่มีบัญชี (Guest)

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/bill-members` | **เจ้าของบิลดึงเพื่อนเข้าบิล** (เพิ่ม Guest/Friend เข้าบิล) |
| `POST` | `/bill-members/join` | **เข้าร่วมบิลผ่าน Code** สำหรับให้ผู้ใช้งานคนอื่นกรอกรหัสบิลเพื่อเข้าร่วมวงหารด้วยตัวเอง |
| `GET` | `/bill-members/:billId`| **ดึงรายชื่อเพื่อนทั้งหมดในบิล** เพื่อเอาไปแสดงรูปโปรไฟล์ในหน้าจอ |
| `PATCH` | `/bill-members/:id/toggle-paid` | **แจ้งโอนเงิน (Toggle)** เพื่อนกดปุ่มว่า "ฉันจ่ายแล้วนะ" สถานะจะเปลี่ยนเป็น `PENDING` รอเจ้าของตรวจ |
| `PATCH` | `/bill-members/:id/verify` | **เจ้าของตรวจสอบ (Verify)** เจ้าของบิลกดยืนยันว่า "ได้รับเงินแล้ว" สถานะจะเปลี่ยนเป็น `PAID` |
| `PATCH` | `/bill-members/:id/submit-slip` | **แนบสลิป (Submit Slip)** เพื่อนส่งสลิปโอนเงินเข้ามาระบบ พร้อมให้ AI ตรวจสอบอัตโนมัติ |

## 4. 🍕 ระบบจัดสรรคนจ่าย (Splits)
**หน้าที่:** กำหนดว่า "ใคร" ต้องจ่าย "เมนูไหน" (ระบบนี้รองรับการหารแบบรายคน แยกจานกัน)

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/splits/assign` | **มอบหมายคนจ่าย (Assign)** กำหนดว่า Item A นาย ก. และ ข. เป็นคนหารร่วมกัน (สามารถโยน Array เข้ามาทีเดียวได้) |
| `GET` | `/splits/:itemId` | **ดูว่าใครหารเมนูนี้บ้าง** เพื่อไฮไลท์รูปโปรไฟล์เพื่อนในเมนูนั้นๆ |

## 5. 📷 ระบบ AI สแกนใบเสร็จ (OCR)
**หน้าที่:** ดึงข้อความจากรูปใบเสร็จแปลงเป็นรายการอาหารอัตโนมัติ 

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/ocr/scan` | **สแกนใบเสร็จด้วย AI (Gemini/Vision)** โยน `imageUrl` ของใบเสร็จเข้าไป ระบบจะประมวลผลกลับมาเป็น Array ของชื่ออาหารและราคา (จำกัด 5 ครั้ง/ชม. กัน Spam) |

## 6. 📁 ระบบอัปโหลดรูปภาพ (Storage)
**หน้าที่:** จัดการไฟล์รูปภาพ เช่น รูปใบเสร็จ หรือ รูปสลิปโอนเงิน

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/storage/upload-url` | **ขอลิ้งค์อัปโหลด (Presigned URL)** (Best Practice) ฝั่ง Frontend ขอ URL เพื่อเอาไปโยนไฟล์เข้า Cloudflare R2 โดยตรง ไม่ให้หนักเซิร์ฟเวอร์ Backend |
| `PUT` | `/storage/upload/:key` | **อัปโหลดเข้า Backend โดยตรง** (ใช้สำหรับตอนรัน Development ในเครื่อง Local เท่านั้น) |
| `GET` | `/storage/file/:key` | **ดึงรูปมาแสดง** (เฉพาะตอน Development ในเครื่อง Local) |

## 7. 🤝 ระบบเพื่อน (Friends)
**หน้าที่:** บันทึกรายชื่อเพื่อนที่ใช้บ่อย เพื่อให้ดึงเข้าวงหารได้ง่ายๆ โดยไม่ต้องพิมพ์ชื่อใหม่ทุกครั้ง

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/friends` | **เพิ่มเพื่อน** บันทึก Guest ที่เจอบ่อยๆ หรือค้นหา User อื่นมาเป็นเพื่อน |
| `GET` | `/friends` | **ดึงรายชื่อเพื่อน** เพื่อแสดงในหน้าเลือกเพื่อนเข้าบิล |
| `PATCH` | `/friends/:id` | **แก้ไขข้อมูลเพื่อน** (เช่น แก้ชื่อเล่น) |
| `DELETE`| `/friends/:id` | **ลบเพื่อน** ออกจาก List |

## 8. 🏦 ระบบบัญชีธนาคาร (Bank Accounts)
**หน้าที่:** ให้เจ้าของบิลผูกบัญชีธนาคารหรือพร้อมเพย์ เพื่อแสดงให้เพื่อนดูตอนโอนเงิน

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/bank-accounts` | **เพิ่มบัญชีรับเงิน** (พร้อมเพย์ หรือ เลขบัญชี) |
| `GET` | `/bank-accounts` | **ดูบัญชีทั้งหมดของตัวเอง** |
| `PATCH` | `/bank-accounts/:id` | **แก้ไขบัญชี** |
| `DELETE`| `/bank-accounts/:id` | **ลบบัญชี** |

## 9. 👤 ระบบผู้ใช้งาน (Users & Webhooks)
**หน้าที่:** ซิงค์ข้อมูลกับระบบ Clerk และจัดการโปรไฟล์

| Method | Endpoint | คำอธิบายโดยละเอียด |
|:---:|:---|:---|
| `POST` | `/users/sync` | **(Internal)** ซิงค์ข้อมูลเมื่อมีคนสมัครสมาชิกใหม่ผ่าน Clerk (Webhook) |
| `GET` | `/users/me` | **ดึงโปรไฟล์ตัวเอง** (ชื่อ, อีเมล, รูปประจำตัว) |
| `PATCH` | `/users/me` | **อัปเดตโปรไฟล์ตัวเอง** |
| `GET` | `/users/search` | **ค้นหาผู้ใช้งานอื่น** เอาไว้แอดเพื่อนในระบบ |
| `POST` | `/webhooks/clerk` | **(Internal)** เส้นทางลับเฉพาะสำหรับให้ระบบเซิร์ฟเวอร์ Clerk ยิงข้อมูลผู้ใช้มาหาเรา |
